# O que é um README.md?

O **leia-me** (geralmente chamado de `README.md`) é o arquivo principal que apresenta o seu projeto no GitHub. Ele funciona como um cartão de visitas ou um manual de instruções: quem entra no seu repositório lê esse arquivo primeiro para entender o que o projeto faz e como usá-lo.

## O que um README costuma incluir

### Título e Descrição
O nome do projeto e qual problema ele resolve.

### Funcionalidades
Os principais pontos fortes e o que o software faz.

### Pré-requisitos e Instalação
Como baixar e configurar o projeto no computador.

### Como usar
Exemplos práticos de uso.

### Contribuição
Como outros desenvolvedores podem ajudar a melhorar o código.

## Linguagem Markdown

Ele é escrito em **Markdown** (por isso a extensão `.md`), uma linguagem simples que permite formatar textos com títulos, negrito, imagens e links.

## README do Perfil do GitHub

Além de documentar projetos, você pode criar um "Leia-me" especial para o seu perfil do GitHub criando um repositório com o seu próprio nome de usuário. É uma ótima maneira de colocar sua biografia, tecnologias que domina e links para redes sociais em destaque para recrutadores.
